# Estructura del proyecto

## Archivos principales

- `index.html`: estructura de la landing.
- `styles.css`: estilos visuales.
- `script.js`: interacciones, carruseles, modales y filtros.
- `assets/images/`: imagenes del hero, productos, escenarios y logos.
- `docs/textos-landing.md`: copy comercial reutilizable.
- `docs/cambios-pendientes.md`: backlog del proyecto.

## Recomendacion de ramas Git

- `main`: version estable publicada.
- `dev`: cambios en progreso.
- `feature/catalogo-limpio`: mejoras del catalogo.
- `feature/mobile-fixes`: ajustes responsive.
- `feature/seo-performance`: optimizacion tecnica.

## Convencion de commits

- `feat: agrega banner principal`
- `fix: corrige overflow en mobile`
- `style: mejora tarjetas de catalogo`
- `docs: actualiza textos de landing`
- `perf: optimiza imagenes`
